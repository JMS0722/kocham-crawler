# 코참(KOCHAM) 디렉토리 크롤러 프로젝트 계획서

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | KOCHAM Directory Crawler |
| 목적 | 코참(KOCHAM) 디렉토리에서 베트남 진출 한국 기업 정보를 수집하여 롯데렌탈 장/단기 렌트카 영업 리드 DB 구축 |
| 산출물 | Windows exe 실행 파일 (GUI) |
| 대상 사용자 | 롯데렌탈 베트남 영업팀 |

## 2. 배경

- 롯데렌탈 베트남 법인의 장/단기 렌트카 상품 영업 대상: 베트남 진출 한국 기업
- 코참(KOCHAM, 호치민한인상공인연합회) 디렉토리에 기업 정보가 등록되어 있음
- 수작업으로 기업 정보를 수집하는 것은 비효율적 → 자동화 크롤러 필요

## 3. 수집 대상 데이터

코참 디렉토리(https://kocham.kr 또는 관련 사이트)에서 수집할 항목:

| 필드 | 설명 |
|------|------|
| 회사명 (국문) | 한국어 회사명 |
| 회사명 (영문/베트남어) | 영문 또는 베트남어 회사명 |
| 업종/카테고리 | 업종 분류 |
| 대표자명 | 대표이사/법인장 |
| 전화번호 | 사무실 연락처 |
| 이메일 | 대표 이메일 |
| 주소 | 베트남 사업장 주소 |
| 웹사이트 | 회사 홈페이지 |

> 실제 사이트 구조 확인 후 필드 조정 필요

## 4. 기술 스택

| 구분 | 기술 | 선정 이유 |
|------|------|-----------|
| 언어 | Python 3.11+ | 크롤링 생태계 우수 |
| HTTP 요청 | requests | 정적 페이지 크롤링 |
| HTML 파싱 | BeautifulSoup4 | 직관적 파싱 |
| 동적 페이지 (필요시) | Selenium / Playwright | JS 렌더링 페이지 대응 |
| 엑셀 출력 | openpyxl | .xlsx 생성 및 스타일링 |
| GUI | tkinter | Python 내장, 추가 설치 불필요 |
| EXE 빌드 | PyInstaller | 단일 exe 빌드, Windows 배포 용이 |

## 5. 주요 기능

### 5.1 크롤링 엔진
- 코참 디렉토리 페이지 자동 탐색
- 카테고리/업종별 필터링 크롤링
- 페이지네이션 자동 처리
- 요청 간 딜레이 (서버 부하 방지, 예의 있는 크롤링)
- 에러 핸들링 및 재시도 로직

### 5.2 데이터 처리
- 수집 데이터 정제 (공백 제거, 포맷 통일)
- 중복 데이터 제거
- 엑셀(.xlsx) 파일 저장
  - 컬럼 자동 너비 조정
  - 헤더 스타일링 (배경색, 볼드)
  - 날짜별 파일명 자동 생성

### 5.3 GUI (tkinter)
- 크롤링 시작/중지 버튼
- 카테고리 선택 (체크박스 또는 드롭다운)
- 진행률 표시 (프로그레스 바 + 로그 창)
- 저장 경로 선택
- 크롤링 완료 시 엑셀 파일 자동 열기 옵션

### 5.4 EXE 배포
- PyInstaller로 단일 exe 빌드
- 아이콘 커스터마이징 (롯데렌탈 또는 커스텀)
- Windows 10/11 호환

## 6. 프로젝트 구조 (예상)

```
kocham-crawler/
├── PROJECT_PLAN.md          # 프로젝트 계획서 (본 문서)
├── requirements.txt         # Python 의존성
├── src/
│   ├── main.py              # 진입점 (GUI 실행)
│   ├── crawler.py           # 크롤링 엔진
│   ├── parser.py            # HTML 파싱 로직
│   ├── exporter.py          # 엑셀 출력
│   └── config.py            # 설정값 (URL, 딜레이 등)
├── assets/
│   └── icon.ico             # EXE 아이콘
├── build.spec               # PyInstaller 빌드 설정
└── dist/                    # 빌드 산출물 (exe)
```

## 8. 주의사항

### 법적/윤리적 고려
- 크롤링 전 robots.txt 확인 필수
- 과도한 요청으로 서버에 부하를 주지 않도록 적절한 딜레이 설정 (최소 1~2초)
- 수집 데이터는 영업 목적 내부 사용에 한정
- 개인정보보호법 준수 (수집 데이터 외부 유출 금지)

### 기술적 고려
- 사이트 구조 변경 시 파서 업데이트 필요
- 로그인 필요 여부 확인 (세션/쿠키 처리)
- 네트워크 불안정 대비 재시도 로직 필수

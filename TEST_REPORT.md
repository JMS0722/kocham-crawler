## 3. 주요 개선 사항 (v2)

| 기능 | 설명 |
|------|------|
| 일시정지/재개 | threading.Event 기반, 크롤링 중 일시정지 후 이어서 수집 |
| 중간저장 | 100건마다 자동 저장 (_autosave 파일), 중지 시에도 중간 결과 보존 |
| 커스텀 파일명 | 사용자가 prefix 입력 가능 (suffix는 자동 날짜) |
| 저장 경로 사전검증 | 크롤링 시작 전 경로 존재 여부 확인 |
| 저장 실패 대응 | 실패 시 "다른 경로 선택" 대화상자 자동 표시 |
| OneDrive 바탕화면 | Windows 레지스트리에서 실제 경로 조회 |
| 경과/남은 시간 | 일시정지 시간 제외한 순수 경과 시간 + ETA 표시 |
| 중복 ID 방지 | set 기반 중복 체크, 새 ID 0건이면 수집 종료 |

## 4. 알려진 제한사항

- POST 필터 크롤링 시 페이지네이션이 서버에서 정상 동작하는지 추가 확인 필요
- 전체 크롤링 (~2,300건) 시 약 40분~1시간 소요 (딜레이 1.5초 기준)
- SSL DH_KEY_TOO_SMALL 우회 적용 (SECLEVEL=1)
